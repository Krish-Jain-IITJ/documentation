About Us

Drona Aviation — building accessible, programmable nano-drones for education and research

Who We Are

Drona Aviation is an Indian deep-tech startup dedicated to making aerial robotics accessible to everyone. Our flagship product, the Pluto drone, is a palm-sized, programmable nano-drone designed for students, educators, researchers, and hobbyists who want to learn about drone technology, autonomous systems, and robotics — all from a safe, affordable platform.

We believe that the best way to learn robotics is to build and fly real hardware. Pluto bridges the gap between simulation and real-world robotics, giving you a physical drone you can program, crash, and learn from — without breaking the bank.

Pluto Nano-Drone

A compact, sturdy nano-drone built for indoor and outdoor experimentation. Weighs under 50g and fits in the palm of your hand.

Fully Programmable

Control Pluto using Python, C++, ROS2, or Drona Aviation's own SDK. Full access to flight commands, sensors, and telemetry data.

Built for Education

Used in universities, coding bootcamps, and research labs across India and beyond. A safe, repeatable platform for drone and AI experiments.

ROS2 Integration

Native ROS2 support (Foxy and later) lets you leverage the full robotics ecosystem — simulation, navigation, sensor fusion, and more.

The Pluto Drone

Pluto is Drona Aviation's flagship nano-drone platform. It is designed from the ground up for programmability and learning:

  • Microcontroller: STM32-based flight controller running Drona Aviation's custom firmware (Magis).
  • Communication: Connects to your laptop or phone over Wi-Fi (acts as a Wi-Fi access point).
  • Sensors: On-board IMU, barometer, and optional range sensors for stable indoor flight.
  • Weight & Size: Ultra-compact nano form factor — safe for indoor use and beginners.
  • SDK & API: Open SDK with support for C, Python, and ROS/ROS2 packages.

ROS2 & Pluto

This documentation focuses on controlling Pluto using ROS2 (Robot Operating System 2). ROS2 is the industry-standard framework for writing robot software, offering a rich ecosystem of tools for communication, simulation, navigation, and autonomy.

By combining Pluto with ROS2, you can:

  • Send flight commands (throttle, roll, pitch, yaw) via ROS2 topics and services
  • Read real-time telemetry data from the drone into your ROS2 nodes
  • Build autonomous behaviours using ROS2's navigation and planning stack
  • Test algorithms in Gazebo simulation before deploying on real hardware
  • Use keyboard, joystick, or your own custom controllers via launch files
Getting started? Head to the Installation section in the sidebar to set up ROS2 Foxy and the Pluto ROS2 package on your Ubuntu machine.

Installation

Set up ROS2 and the Pluto drone package on your system

Caution: Please pay close attention to the version of software you're using. The dependencies change heavily between distributions.

Prerequisites

  • Ubuntu 20.04 or a greater version
  • Internet connection for downloading packages
  • A Pluto drone (without camera module) and a laptop connected to the same Wi-Fi
1 Install ROS2 Foxy & Setup the Environment

Follow the official ROS2 Foxy installation guide for Ubuntu. Use the Development Setup path for best compatibility:

https://docs.ros.org/en/foxy/Installation/Alternatives/Ubuntu-Development-Setup.html

Install Ubuntu 20.04 or greater before this step.
2 Create a New Workspace Directory
bash
$ mkdir -p ~/ros2_ws/src
$ cd ~/ros2_ws/src
3 Install Git
bash
$ sudo apt-get install git
4 Install the Pluto ROS2 Package

Navigate to the src folder of your workspace:

bash
# Navigate to src folder
$ cd ~/ros2_ws/src

# Clone the Pluto ROS2 package
$ git clone https://github.com/DronaAviation/pluto-ros2-package

# Build the package
$ colcon build

# Source workspace into bashrc
$ echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
5 Test Run the Pluto Package

Open three separate terminals. Make sure ROS2 is sourced in every terminal, and that your Pluto Drone (without the camera) is powered on and your laptop is connected to its Wi-Fi network.

Terminal 1 — Start the Pluto Node

Terminal 1
$ ros2 run plutodrone plutonode
Pluto Connected

Terminal 2 — Start the Pluto Service Node

Terminal 2
$ ros2 run plutoserver plutoserver

Terminal 3 — Launch Keyboard Control

Terminal 3
$ ros2 launch plutoserver drone_comb.launch.py
For the full list of keyboard control commands, see Section 4.2 of the ROS wiki tutorial.

Setup

Configure the Pluto dashboard environment after installation

Environment Configuration

After completing the Installation steps, use the following setup guide to configure your environment for the Pluto ROS2 dashboard and development workflow.

1 Verify ROS2 Environment

Open a terminal and confirm ROS2 is correctly sourced:

bash
$ echo $ROS_DISTRO
foxy

$ ros2 --version
ros2cli 0.9.x
2 Source the Workspace on Every Shell

If you haven't already added the source command to your .bashrc, do it now so every new terminal is ready:

bash
$ source ~/.bashrc
# Or add permanently:
$ echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
3 Wi-Fi Connection to Pluto Drone

Power on your Pluto drone and connect your laptop to the drone's Wi-Fi hotspot. Verify connectivity by pinging the drone's default IP:

bash
$ ping 192.168.4.1
64 bytes from 192.168.4.1: icmp_seq=1 ttl=64 time=1.2 ms
The Pluto drone acts as a Wi-Fi access point. Make sure no other Wi-Fi network is interfering.
4 Rebuild after Any Changes

Whenever you modify source files in the package, rebuild from the workspace root:

bash
$ cd ~/ros2_ws
$ colcon build
$ source install/setup.bash
5 Confirm Package is Listed

Verify the Pluto package is visible to ROS2:

bash
$ ros2 pkg list | grep pluto
plutodrone
plutoserver
If the packages are not listed, re-run source ~/ros2_ws/install/setup.bash and try again. Make sure colcon build completed without errors.

Teleop & Starting Commands

Follow these steps to connect your Pluto drone to the PC server using ROS and initiate teleoperation.

6 Start Pluto Server

Connect the drone to the server.

bash
$ ros2 run plutoserver plutoserver
7 Start Teleop

Launch the keyboard teleoperation node to control the drone manually.

bash
$ ros2 run plutodrone pluto_teleop

Dashboard

Connect Pluto commands to the dashboard interface for advanced monitoring and control

Dashboard Integration

Use the following commands to link the Pluto drone with your visual dashboard.

1 Launch Dashboard Node

Start the ROS node responsible for the dashboard interface.

bash
$ ros2 run pluto_dashboard dashboard_node

ArUco Marker Establishment

For complex autonomous tasks, you can use ArUco markers for positioning, tracking, and precision landing.

1 Place Markers

Print and place ArUco markers in your physical environment. Ensure they are flat and well-lit. Measure and note the exact physical dimensions of the printed markers (e.g., 5cm x 5cm) as this is crucial for accurate distance estimation.

2 Start Camera Node

Enable the drone's camera stream to detect the markers.

bash
$ ros2 run pluto_camera camera_node
3 Start Tracking Node

Launch the ArUco tracking script to estimate pose and execute autonomous instructions.

bash
$ ros2 run pluto_vision aruco_tracker

Simulate

Run the 2D Control Simulation for Pluto

Running the Pluto Sim Window

Before flying the real drone, you can test the path following capabilities and control parameters using the built-in 2D simulation window.

1 Run in PowerShell (Windows)

If you are developing on Windows, open your PowerShell terminal, navigate to the project directory, and execute the python script.

powershell
PS> cd C:\path\to\Drona_aviation
PS> python sim_window.py
2 Run in Ubuntu Terminal (Linux/ROS2 environment)

If you are running in your Ubuntu environment (like when using ROS2), open a standard bash terminal and execute the following commands:

bash
$ cd ~/path/to/Drona_aviation
$ python3 sim_window.py

Launch

Start the Pluto dashboard across different operating systems

Launch Instructions

Follow the steps below to set up and launch the Pluto dashboard. Select your operating system for the appropriate commands.

1 Install RoboStack ROS 2 Humble

One-time setup: install RoboStack ROS 2 Humble into a conda env called "ros2".

macOS / Ubuntu Bash

bash
$ ./bootstrap_ros.sh

PowerShell

powershell
PS> .\bootstrap_ros.ps1
2 Clone the Repositories

Clone the main fork and the camera fork side-by-side.

macOS / Ubuntu Bash

bash
$ mkdir -p ~/pluto && cd ~/pluto
$ git clone -b dashboard-integration https://github.com/techsavvyomi/pluto_ros2_package.git
$ cd pluto_ros2_package
$ git clone -b fix-1080p-framebuffer https://github.com/techsavvyomi/pluto_cam_ros2.git pluto_ros2_package/pluto_cam_ros2

PowerShell

powershell
PS> mkdir ~/pluto; cd ~/pluto
PS> git clone -b dashboard-integration https://github.com/techsavvyomi/pluto_ros2_package.git
PS> cd pluto_ros2_package
PS> git clone -b fix-1080p-framebuffer https://github.com/techsavvyomi/pluto_cam_ros2.git pluto_ros2_package/pluto_cam_ros2
3 Install Camera Dependencies

Install camera dependencies inside the conda environment.

bash / powershell
$ conda activate ros2
$ mamba install -y -c conda-forge -c robostack-staging ros-humble-cv-bridge ros-humble-image-transport opencv av
4 Build the Workspace

macOS / Ubuntu Bash

bash
$ ./build.sh

PowerShell

powershell
PS> .\build.ps1
5 Launch the Dashboard

Launch just the dashboard — the only terminal you need.

macOS / Ubuntu Bash

bash
$ ./start_dashboard.sh

PowerShell

powershell
PS> .\start_dashboard.ps1
