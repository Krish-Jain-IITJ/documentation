What is This?

This documentation guides you through 10 progressive projects (P0–P9) to go from a bare WSL2 install all the way to autonomous ArUco-marker hovering with the PlutoDrone nano-drone using ROS2 Jazzy on Windows 11 WSL2.

System Architecture

Windows 11 + WSL2

Ubuntu 24 running inside WSL2. ROS2 Jazzy installed natively. WiFi bridged to drone AP.

PlutoDrone

STM32 flight controller. MSP protocol over WiFi at 192.168.0.1:9060. Optional camera module.

pluto_ros2_package

GitHub repo by techsavvyomi. Contains plutodrone, plutoserver, pluto_dashboard, pluto_vision, pluto_camera_sense.

Dashboard

Flask + rclpy web UI at localhost:5050. Arm/takeoff/land, telemetry, camera preview, process manager.

PROJECT 0

ROS2 Installation & WSL2 Environment Setup

Get ROS2 Jazzy running inside WSL2 on Windows 11 and clone the Pluto package

All commands run inside WSL2 Ubuntu 24 terminal unless specified otherwise.

Step 1 — Enable WSL2 & Install Ubuntu 24

1 Enable WSL2 in PowerShell (Run as Admin)
PowerShell (Admin)
wsl --install -d Ubuntu-24.04
# Restart Windows when prompted

Step 2 — Install ROS2 Jazzy inside WSL2

2 Add ROS2 apt repository
WSL2 bash
sudo apt update && sudo apt install -y software-properties-common curl
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list
3 Install ROS2 Jazzy Desktop
WSL2 bash
sudo apt update
sudo apt install -y ros-jazzy-desktop python3-colcon-common-extensions python3-pip
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
4 Verify Installation
WSL2 bash
ros2 --version
ros2cli 0.18.x (jazzy)

Step 3 — Clone & Build Pluto Package

5 Clone the repository
WSL2 bash
mkdir -p ~/pluto_ws && cd ~/pluto_ws
git clone -b dashboard-integration https://github.com/techsavvyomi/pluto_ros2_package.git
cd pluto_ros2_package
# Also clone camera fix
git clone -b fix-1080p-framebuffer https://github.com/techsavvyomi/pluto_cam_ros2.git pluto_ros2_package/pluto_cam_ros2
6 Install dependencies & build
WSL2 bash
cd ~/pluto_ws/pluto_ros2_package
pip install av numpy --break-system-packages
sudo apt install -y ffmpeg python3-colcon-common-extensions
source /opt/ros/jazzy/setup.bash
colcon build
echo "source ~/pluto_ws/pluto_ros2_package/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
ROS2 Jazzy + Pluto package is now installed. Proceed to Project 1.

Drone Connection & Telemetry

Connect PlutoDrone over WiFi and read live IMU, battery, and altitude topics

Connect your PC WiFi to the PlutoDrone WiFi hotspot before running any commands.

Verify Network

Start Pluto Node

2 Terminal 1 — Launch plutonode
Terminal 1

ros2 run plutodrone plutonode

Read Telemetry Topics

3 Terminal 2 — View live telemetry
Terminal 2
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash

# List all pluto topics
ros2 topic list | grep pluto

# Read IMU data
ros2 topic echo /pluto/imu

ros2 topic echo /pluto/battery

ros2 topic echo /pluto/altitude

Pluto RViz Visualisation

Visualise drone orientation, TF frames, and telemetry in RViz2

RViz requires a display. Use WSLg (Windows 11 21H2+) or set DISPLAY manually.

Setup WSLg Display

1 Verify WSLg is working
WSL2 bash
echo $DISPLAY

# If empty, add to ~/.bashrc:
export DISPLAY=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}'):0

Launch Dashboard + RViz

2 Terminal 1 — Start plutonode
Terminal 1
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash

3 Terminal 2 — Launch TF broadcaster + RViz
Terminal 2
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash

# In Terminal 3 — open RViz
rviz2

  • Add → Imu → topic: /pluto/imu

Terminal 3

Keyboard Controls

Flight Commands

  • T — Takeoff / Arm
  • L — Land / Disarm
  • W / S — Throttle up / down
  • A / D — Yaw left / right

Movement

  • ↑ / ↓ — Pitch forward / back
  • ← / → — Roll left / right
  • Space — Emergency stop
  • H — Hover / altitude hold
Full keyboard reference: wiki.ros.org/pluto_drone#Keyboard

PROJECT 6

Hand Gesture Detection

Use MediaPipe to detect hand gestures from drone camera and control flight

How It Works

The hands_worker.py module uses MediaPipe Hands to detect hand landmarks from JPEG frames received over a binary stdin/stdout pipe. It runs in a separate Python subprocess (inside a mediapipe conda env with NumPy 1.x) to avoid conflicts with the ROS2 dashboard process (NumPy 2.x).

Setup MediaPipe Environment

1 Create isolated Python env for MediaPipe
WSL2 bash
python3 -m venv ~/mediapipe_env
source ~/mediapipe_env/bin/activate
pip install mediapipe opencv-python "numpy<2"
deactivate

Start the Stack

2 Run plutonode + camera (Terminals 1 & 2)

Start plutonode and plutocam_publisher --low-def as in Project 5 first.

3 Launch Dashboard with hand detection
Terminal 3
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
ros2 run pluto_dashboard dashboard_node
# Open browser: http://localhost:5050
# Enable "Hand Gesture" in Services panel

Gesture Reference

Recognized Gestures

  • Open palm — Hover
  • Fist — Land
  • One finger up — Takeoff
  • Peace sign — Rotate

Protocol (stdin/stdout)

  • Parent → Worker: 4-byte uint32 length + JPEG bytes
  • Worker → Parent: 4-byte uint32 length + annotated JPEG
  • Runs in ~/mediapipe_env subprocess
The hands worker returns the original JPEG unchanged if detection fails for a frame — so the video feed never breaks.
PROJECT 7

ArUco Detection

Detect ArUco markers from the drone camera feed using OpenCV

Print ArUco Markers

1 Generate a 4x4 ArUco marker (ID 0)
WSL2 bash
pip install opencv-python --break-system-packages
python3 -c "
import cv2
d = cv2.aruco.getPredefinedDictionary(cv2.aruco.DICT_4X4_50)
img = cv2.aruco.generateImageMarker(d, 0, 300)
cv2.imwrite('/tmp/aruco_0.png', img)
print('Saved to /tmp/aruco_0.png')"
# Copy to Windows and print at exactly 10cm x 10cm

Start Detection Node

2 Start plutonode + camera (as in P5)

Terminals 1 and 2: plutonode and plutocam_publisher --low-def

3 Terminal 3 — Run ArUco detection node
Terminal 3
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
ros2 run pluto_vision aruco_follow --ros-args -p enabled:=false
# enabled:=false = detection only, no flight commands
4 Verify detection output
Terminal 4
ros2 topic echo /pluto/aruco_detection
# Shows marker ID, corners, and estimated pose

Detection Parameters

  • Dictionary: DICT_4X4_50 (IDs 0–49)
  • Subscribes to: /plutocamera/image_raw
  • Default: disabled — enable with -p enabled:=true
  • Gain parameter: gain_px_to_rc (default 1.5)
PROJECT 8

ArUco Tracking

Automatically follow an ArUco marker using proportional roll/pitch control

The drone will move autonomously. Ensure a clear 2m x 2m space. Keep a thumb on the disarm key.

How the Tracker Works

The ArucoFollow node subscribes to /plutocamera/image_raw, detects the marker center, computes pixel error from frame center, and publishes proportional roll/pitch corrections to /drone_command. Throttle stays neutral — the firmware handles altitude hold.

Launch Tracking

1 Start plutonode + camera (Terminals 1 & 2)

Start plutonode and plutocam_publisher --low-def first as in Project 5.

2 Terminal 3 — Enable ArUco follower
Terminal 3
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
ros2 run pluto_vision aruco_follow \
--ros-args \
-p enabled:=true \
-p gain_px_to_rc:=1.5 \
-p dict_id:=0
3 Arm and takeoff (Terminal 4)
Terminal 4
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
# Arm + takeoff to stable altitude
ros2 topic pub /drone_command custom_msgs/msg/PlutoMsg \
"{throttle: 1600, roll: 1500, pitch: 1500, yaw: 1500, aux1: 1500, aux2: 1000}" \
--once
# Now hold the ArUco marker in view — drone will follow it

Tuning

  • Increase gain_px_to_rc for faster response (risk: overshoot)
  • Decrease for slower, smoother tracking
  • RC range: 1000–2000, neutral at 1500
  • Pixel error at frame edge ≈ ±640px → maps to ±960 RC units at default gain
PROJECT 9

ArUco Hovering

Maintain precise position above an ArUco marker using altitude + position hold

Advanced project — requires stable camera feed, tuned PID, and altitude hold enabled on firmware.

How Hovering Works

ArUco hovering combines lateral tracking (P8) with altitude control using the barometer. The node estimates marker distance from apparent size to maintain a fixed height above the marker, while roll/pitch keep it centered.

Prerequisites

  • Project 5 camera feed working at stable 25 Hz
  • Project 8 tracking working (marker centering)
  • Altitude hold enabled in Pluto firmware (aux channel)
  • Marker physical size measured and configured (e.g. 10cm)

Launch Full Hover Stack

1 Terminal 1 — plutonode
Terminal 1
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
ros2 run plutodrone plutonode
2 Terminal 2 — Camera publisher
Terminal 2
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
plutocam_publisher --low-def
3 Terminal 3 — Altitude hold node
Terminal 3
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
ros2 run pluto_dashboard pid_altitude_hold
4 Terminal 4 — ArUco follower (enabled)
Terminal 4
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
ros2 run pluto_vision aruco_follow \
--ros-args \
-p enabled:=true \
-p gain_px_to_rc:=1.2
5 Terminal 5 — Arm & set altitude hold mode
Terminal 5
source /opt/ros/jazzy/setup.bash
source ~/pluto_ws/pluto_ros2_package/install/setup.bash
# Arm drone with altitude hold (aux1=2000)
ros2 topic pub /drone_command custom_msgs/msg/PlutoMsg \
"{throttle: 1500, roll: 1500, pitch: 1500, yaw: 1500, aux1: 2000, aux2: 1000}" \
--rate 20
# Place ArUco marker below drone — it will hold position above it

Expected Behavior

  • Drone takes off and holds altitude via barometer PID
  • Camera detects ArUco marker below/in front
  • Roll/pitch corrects to keep marker centered in frame
  • Move the marker — drone follows and re-centers
  • Remove marker from view — drone returns to neutral hover
Tip: Start with gain_px_to_rc:=0.8 for very gentle response. Increase slowly until tracking feels crisp without oscillation.
